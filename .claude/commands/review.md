# /review

**描述**: 全面代码审查，发现潜在问题
**哲学**: Code Review - 提升代码质量和可维护性

---

## 用法

```
/review                   # 审查当前文件
/review filename.R        # 审查指定文件
/review --focus=security  # 专注安全审查
/review --focus=performance  # 专注性能审查
```

---

## 审查维度

### 1. 安全性 🔒
- 注入漏洞（SQL、命令注入）
- 敏感信息泄露
- 不安全的文件操作

### 2. 性能 ⚡
- 不必要的循环
- 内存泄漏风险
- 低效的数据结构

### 3. 可靠性 🛡️
- 错误处理缺失
- 边界情况未处理
- 资源未释放

### 4. 可维护性 🔧
- 硬编码值
- 魔法数字
- 缺乏文档

```
## 🔍 代码审查报告

- [ ] 问题描述 | 行号 | 风险等级

### 性能
- [ ] 问题描述 | 行号 | 影响程度

```代码示例```
```

## 严重级别

| 图标 | 级别 | 说明 |
|------|------|------|
| 🔴 | Critical | 必须立即修复 |
| 🟠 | High | 尽快修复 |
| 🟡 | Medium | 建议修复 |
| 🟢 | Low | 可选改进 |
