# /simplify

检查代码是否符合 Linux 极简主义哲学（Unix Philosophy: Do one thing well）

## 审查清单

1. **单一职责** - 每个函数/模块是否只做一件事？
2. **函数长度** - 函数是否短小精悍（建议 ≤30 行）？
3. **嵌套深度** - 控制流嵌套是否 ≤3 层？
4. **早返回** - 是否使用卫语句（guard clauses）减少嵌套？
5. **无冗余** - 是否有不必要的变量、注释或重复代码？
6. **命名清晰** - 变量/函数名是否自解释？
7. **行宽适中** - 是否 ≤100 字符，便于阅读？

## 输出要求

请审查当前文件或最近修改的代码：

如有改进空间，请：
1. 指出具体问题（行号+原因）
2. 给出简化后的代码示例
3. 解释为什么这样更简洁

如代码已足够简洁，请回复：
> ✅ 代码符合 Linux 极简主义原则
