# /simplify

**描述**: 检查并简化代码，遵循 Linux 极简主义哲学
**哲学**: Unix Philosophy - Do one thing well

---

## 用法

```
/simplify                  # 检查当前打开的文件
/simplify filename.R       # 检查指定文件
/simplify --dry-run        # 只检查，不修改（默认）
/simplify --apply          # 原地修改文件
```

---

## 工作流程

1. **读取文件** → 分析代码结构和质量
2. **识别问题** → 对照审查清单检查
3. **生成报告** → 指出具体问题和改进方案
4. **简化代码** → 如指定 `--apply`，直接原地修改

---

## 审查清单

| 检查项       | 标准                | 严重程度 |
| ------------ | ------------------- | -------- |
| **单一职责** | 每个函数只做一件事  | 🔴 严重   |
| **函数长度** | ≤30 行              | 🔴 严重   |
| **嵌套深度** | ≤3 层               | 🔴 严重   |
| **早返回**   | 使用卫语句减少嵌套  | 🟡 中等   |
| **无冗余**   | 无重复代码/冗余变量 | 🟡 中等   |
| **命名清晰** | 自解释的变量/函数名 | 🟡 中等   |
| **行宽适中** | ≤100 字符           | 🟢 轻微   |

## 原地修改规则

当用户使用 `/simplify --apply` 或明确要求"原地修改"时：

1. **直接修改文件** - 使用 Write/Edit 工具更新原文件
2. **保持功能不变** - 简化代码但不改变行为
3. **分步骤修改** - 复杂修改分多次 Edit 完成
4. **验证结果** - 修改后读取文件确认正确
